# kmp-general (AI compact)

General KMP project template. Targets Android, iOS, JVM. Compose Multiplatform UI.

## Targets

Android (library), iosArm64, iosSimulatorArm64, jvm.

## Source Sets

| Set | Contents |
|-----|----------|
| `commonMain/` | Shared UI (`App.kt`), domain, `expect fun platformName()` |
| `androidMain/` | `MainActivity`, platform actuals |
| `iosMain/` | `MainViewController`, platform actuals |
| `jvmMain/` | `fun main()` for desktop, platform actuals |
| `commonTest/` | kotest matchers, coroutines test |

## Key Deps

Compose Multiplatform (runtime, foundation, material3, desktop-common), kotlinx-serialization-json, kotlinx-coroutines, kotest-assertions.

## Patterns

- `expect`/`actual` only for platform APIs (here: `platformName()`). Prefer interface-based abstraction for complex deps.
- No `android.*` or `java.*` in `commonMain`.
- Manual DI via constructor injection.

## Adding an App Shell

```kotlin
// :android module — depends on shared lib
// AndroidManifest.xml with <activity android:name=".MainActivity">
// build.gradle.kts: application plugin + implementation(project())
```

For iOS, create an Xcode project targeting the KMP framework.
